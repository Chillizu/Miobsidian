## RSA

### 实现与时间复杂度 / Implementation & Time Complexity

```
加密 Encryption:   c = m^e mod n      — 一次模幂 / one modular exponentiation
解密 Decryption:   m = c^d mod n      — 一次模幂 / one modular exponentiation
密钥生成 KeyGen:   找大素数 p,q, 计算 e,d  — 素数测试 + 扩展欧几里得 / primality test + extended Euclid
```

**时间复杂度 $O((\log n)^3)$:**
模幂运算使用"平方-乘"算法（二进制展开指数），每次乘法 $O((\log n)^2)$，指数有 $\log n$ 位 → 总共 $O((\log n)^3)$。密钥越长，指数位数和乘法规模线性增长，总成本立方增长。

### 群关系 / Group Relation

RSA 工作在 $(\mathbb{Z}/n\mathbb{Z})^\times$（模 $n$ 的乘法群）上：
- $m$ 必须是群中的元素（即 $\gcd(m, n) = 1$）
- 公钥 $(n, e)$：加密就是计算群中的 $e$ 次幂 $m^e$
- 私钥 $d$：解密就是计算群中的 $d$ 次幂 $c^d$
- 安全性依赖：不知道群阶 $\varphi(n) = (p-1)(q-1)$ 就无法算出 $d$
- **一句话**：加密"乘 $e$ 次"（群幂），解密"乘 $d$ 次"（群幂），因为群阶未知所以外人无法反推 $d$。

## DH (Diffie-Hellman)

### 实现与时间复杂度 / Implementation & Time Complexity

```
Alice → Bob:  A = g^a mod p     — 一次模幂
Bob   → Alice: B = g^b mod p     — 一次模幂
双方 Shared:  K = g^(ab) mod p    — 一次模幂（各自计算）
```

一次握手 = 2 次模幂交换。密钥协商完成后不再有加密运算（由派生密钥处理后续对称加密）。

### 群关系 / Group Relation

DH 工作在一个大素数阶循环子群 $\langle g \rangle \subset (\mathbb{Z}/p\mathbb{Z})^\times$ 上：
- $g$ 是生成元，阶为 $q$（大素数）
- Alice 的秘密 $a$ 是群上的标量（指数），公开 $A = g^a$ 是群元素
- 双方计算 $K = g^{ab}$：$A^b = (g^a)^b = g^{ab} = B^a = (g^b)^a$
- 安全依赖 DLP 困难性：知道 $g^a$ 和 $g$ 无法在多项式时间内找出 $a$
- **一句话**：把标量（秘密）映射成群元素（公开），利用群幂的可交换性共享密钥，DLog 保证映射不可逆。

## ElGamal

### 实现与时间复杂度 / Implementation & Time Complexity

```
公钥 PK:  h = g^x mod p                      — 一次模幂
加密 Enc:  c1 = g^y, c2 = m · h^y mod p     — 两次模幂
解密 Dec:  m = c2 · c1^(-x) mod p           — 一次模幂
```

加密比 RSA 多一次模幂（产生随机 $y$），密文膨胀一倍（$(c_1, c_2)$）。

### 群关系 / Group Relation

同 DH 群结构 $\langle g \rangle \subset (\mathbb{Z}/p\mathbb{Z})^\times$：
- 加密：用接收方公钥 $h = g^x$，挑随机 $y$，计算共享秘密 $h^y = g^{xy}$，然后用它掩盖消息 $m$
- 解密：用自己的秘密 $x$ 恢复 $g^{xy} = c_1^x$，再除去掩盖
- **一句话**：DH 密钥协商的原位加密版——先"协商"共享秘密（$g^{xy}$），用它遮掩消息。

## DSA

### 实现与时间复杂度 / Implementation & Time Complexity

```
签名 Sign:    r = (g^k mod p) mod q     — 一次模幂
             s = k^(-1)(H(m) + xr) mod q
验证 Verify:  验证两个群元素等式           — 两次模幂
```

签名产生两个整数 $(r, s)$；验证需计算 $u_1 = H(m) \cdot s^{-1} \bmod q$、$u_2 = r \cdot s^{-1} \bmod q$，然后检查 $r \equiv (g^{u_1} y^{u_2} \bmod p) \bmod q$。显著特点是双模数（$p$ 和 $q$），效率略低于 RSA 签名。

### 群关系 / Group Relation

同样 $\langle g \rangle \subset (\mathbb{Z}/p\mathbb{Z})^\times$，$q = \mathrm{ord}(g)$：
- 签名：秘密 $x$ → 群元素 $y = g^x$（公钥）；签名用随机 $k$ 产生群元素 $g^k$
- 验证：验证 $g^{H(m) \cdot s^{-1}} \cdot y^{r \cdot s^{-1}} = g^k$，即验证群中等式
- **一句话**：用群幂构造一个"知道秘密 $x$ 才能产生的等式"，验证方只验等号，不碰秘密。

## ECC (Elliptic Curve Cryptography)

### 实现与时间复杂度 / Implementation & Time Complexity

```
标量乘 Scalar mult: Q = kP    — 椭圆曲线上点的"加法-倍加"
ECDH:   A = aG, B = bG, K = aB = bA = abG
ECDSA:  签名/验证与 DSA 类似，但运算在 E(F_q) 上
```

**时间复杂度 $O((\log q)^3)$:**
标量乘使用 double-and-add（类似平方-乘），每次点加/倍加都是有限域 $\mathbb{F}_q$ 上的运算（域乘 $O((\log q)^2)$，域逆代价更高）。$\log q$ 个比特 → 总成本 $O((\log q)^3)$。

**为什么相同密钥长度下 ECC 比 RSA 安全得多？**
对 RSA 的 GNFS 攻击是亚指数 $L_n[1/3, c]$；对 ECC 的 Pollard $\rho$ 攻击是指数 $O(\sqrt{q}) = O(2^{n/2})$。所以在同等安全强度下 ECC 密钥长度远小于 RSA（256-bit ECC ≈ 3072-bit RSA）。

### 群关系 / Group Relation

$E(\mathbb{F}_q)$ 是全体 $(x, y) \in \mathbb{F}_q^2$ 满足 $y^2 = x^3 + ax + b$ 的点加上一个无穷远点 $O$，构成 Abel 群：
- 群律：弦切弦法（chord-tangent）— 两点连线与曲线相交，取对称点
- 加密（ECDH）：标量乘 $aP$ 是 $P$ 在群上加 $a$ 次
- 安全性：ECDLP 比 DLP 更难 — 已知 $P$ 和 $aP$ 求 $a$
- **一句话**：把 DLP 搬到椭圆曲线上，只有"标量乘"没有"指数幂"，DLog 攻击复杂度从亚指数升到指数，同样比特数安全得多。

## AES

### 实现与时间复杂度 / Implementation & Time Complexity

```
每轮 Each round:
  SubBytes    — 查 S-box (GF(2^8) 逆元 + 仿射变换)
  ShiftRows   — 字节移位（按索引重排）
  MixColumns  — GF(2^8) 矩阵乘法（固定矩阵）
  AddRoundKey — XOR 密钥异或
```

所有操作都在 $\mathrm{GF}(2^8)$ 上进行。S-box 可以预计算为 256 字节查表，MixColumns 是固定矩阵乘法（也可查表）。每块数据（16 字节）的运算次数固定，与密钥长度无关 → **$O(1)$ 每块**。

AES-128 用 10 轮，AES-192 用 12 轮，AES-256 用 14 轮——轮数随密钥增长，但仍是常数。

### 群关系 / Group Relation

AES 使用两层群结构：

**加法群 $(\mathrm{GF}(2^8), +)$:**
- 字节 = $\mathbb{F}_2$ 上的 8 维向量
- XOR 就是向量加法
- SubBytes 中的逆元运算 + AddRoundKey 的 XOR 都发生在此群

**乘法群 $(\mathrm{GF}(2^8)^\times, \cdot)$:**
- 非零字节构成 255 阶循环群
- SubBytes 计算 $x^{-1}$ 在此群中进行
- 列混合 MixColumns 则是将字节视为 $\mathrm{GF}(2^8)$ 上的多项式进行乘法

- **一句话**：AES 的"代数结构"是 $\mathrm{GF}(2^8)$——字节是域元素，加法是 XOR，乘法和逆元都在这个域里完成。这个结构使得 AES 能用代数描述，但目前没有已知的代数攻击能比穷举更快。

## DES

### 实现与时间复杂度 / Implementation & Time Complexity

```
每轮 Each round (16 rounds):
  扩展置换 Expansion (32→48 bits)
  XOR 子密钥
  S-box 替换 (6→4 bits, 8个S-box)
  P置换
  XOR 左半
```

所有操作都是位置换和 S-box 查表，只有 XOR 涉及简单代数运算。$O(1)$ 每块。56 位密钥 → 穷举 $2^{56}$ 步，现代硬件可在数小时内暴力破解。

### 群关系 / Group Relation

**关键结论：DES 的全体加密映射 $\{E_K\}$ 在复合下不构成群。**

Campbell & Wiener (1992) 证明：
- $E_{K_1} \circ E_{K_2}$ 不等于任何单一 $E_{K_3}$
- DES 加密函数生成一个规模 $\geq 10^{2499}$ 的子群
- 这意味着**三重 DES**（3DES: $E_{K_1} \circ D_{K_2} \circ E_{K_3}$）确实增加了安全性，而不是退化为单层 DES

- **一句话**：DES 每轮只是置换 + 查表，$\mathrm{GF}(2)$ 上的代数极少。它的非群性是偶然（设计没有考虑代数封闭性），但正好防止了"双重加密退化"的问题。

## Hash Functions (MD5, SHA-1, SHA-2)

### 实现与时间复杂度 / Implementation & Time Complexity

```
Merkle-Damgard 迭代:
  H_0 = IV (初始向量)
  H_i = f(H_{i-1}, M_i)     — 压缩函数
  输出 H_n

SHA-256:
  每块 64 轮操作（位运算 + 模加 + 逻辑函数）
  ⊕, ∧, ∨, 循环移位 + 模 2^32 加法
```

压缩函数 $f$ 使用大量位运算（XOR、AND、OR、循环移位）和模加法。所有操作在 $\mathbb{F}_2$（位向量）和 $\mathbb{Z}/2^{32}\mathbb{Z}$（模加）上进行。每块常数轮 → **$O(1)$ 每块**。

### 群关系 / Group Relation

**Hash 函数不是群结构。** Merkle-Damgard 构造是一个迭代压缩过程：
- 没有"加密/解密"配对——只有单向压缩
- 没有公开/秘密密钥对
- 安全要求是抗碰撞、抗原像，不是群运算的不可逆性
- 压缩函数 $\{0,1\}^m \to \{0,1\}^n$（$m > n$）是**非双射**的，不可能构成群

- **一句话**：Hash 不依赖群的单向性，而是依赖压缩函数的"单向 + 抗碰撞"。位运算和模加只提供扩散和混淆，没有群论层面的安全论证。

## 总结 / Summary

| 结构类型 | 算法 | 核心群运算 | 为什么是这个复杂度 |
|---|---|---|---|
| **有限域乘法群** | RSA, DH, ElGamal, DSA | 模幂 $g^a \bmod p$ | 平方-乘 $O((\log p)^3)$ → 亚指数安全 |
| **椭圆曲线点群** | ECC | 标量乘 $kP$ | double-and-add $O((\log q)^3)$ → 指数安全更优 |
| **有限域加法+乘法** | AES | GF(2^8) 逆元 + 矩阵乘 | 固定轮数 → $O(1)$，密钥可安全使用到量子时代 |
| **置换集（非群）** | DES | 位置换 + XOR | $O(1)$ 但 56 位密钥太短 → 已淘汰 |
| **非群（压缩迭代）** | MD5, SHA-1, SHA-256 | XOR + 循环移位 + 模加 | $O(1)$，安全性来自压缩函数的设计强度 |
