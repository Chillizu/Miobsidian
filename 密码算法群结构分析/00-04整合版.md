## 1. RSA 加密法

**一句话原理**：一种「公开一把锁、自己藏一把钥匙」的加密法。靠一个数学上的「单向门」——乘法容易、分解质因数极难。你把两个大质数乘起来得到 `n` 公开出去，从 `n` 拆回那两个质数在计算上几乎不可能，这就是安全根基。

**两个核心概念**
- **公钥 (n, e)**：公开给所有人，用来加密。
- **私钥 (n, d)**：只有自己留着，用来解密。

**一个小例子（极小数字，便于手算）**
- 选质数 `p=3, q=11` → `n = 33`，`φ(n) = 20`
- 选公钥 `e=3`（与 20 互质）
- 算私钥 `d=7`（因为 `3×7 = 21 ≡ 1 mod 20`）
- 朋友发消息 `m=4`：加密 `c = 4³ mod 33 = 64 mod 33 = 31`
- 你解密：`m = 31⁷ mod 33 = 4` ✅

**用在哪**：HTTPS 网页加密、加密通信、数字签名（证明消息确实是你发的）。

## 2. 数字签名（RSA 的另一用法）

把 RSA「倒过来」用：**你用私钥对消息签名，别人用你的公钥验证。** 因为私钥只有你一个人有，能还原出原消息就证明：① 消息确实是你发的；② 没被篡改。

**流程**
1. 签名 `sig = doc^d mod n`
2. 发送「消息 + 签名」
3. 验证 `sig^e mod n == 消息？`

**例子**：`doc=4` → `sig = 4⁷ mod 33 = 16`；验证 `16³ mod 33 = 4` ✓

> 真实场景里**不直接对原文签名**，而是先对消息算**哈希值**再对哈希签名——签名短、验证快，且能同时校验完整性。

## 3. 密码类型一览

| 大类 | 代表 | 钥匙 | 用途 |
|------|------|------|------|
| 非对称（群基） | RSA, ECC, DH, ElGamal, DSA | 公钥 + 私钥 | 密钥交换、签名、加密 |
| 对称 | AES（DES 淘汰） | 同一把 | 大量数据加密 |
| 哈希函数 | SHA-256（MD5 弃用） | 无 | 指纹、完整性、签名前压缩 |
| KDF | PBKDF2, Argon2 | 输入秘密 | 从密码 / 秘密派生密钥 |

**各算法一句话**
- **RSA**：乘法群上的指数运算，安全靠大数分解。
- **ECC**：椭圆曲线点群，密钥短、强度高。
- **DH（Diffie–Hellman）**：只做密钥交换，不加密内容；安全靠离散对数 (DLP)。
- **ElGamal**：把 DH 思路改造成加密。
- **DSA**：专门签名（ElGamal 签名变体），美国官方标准。
- **AES**：现行对称标准，在有限域 GF(2⁸) 上运算。
- **DES**：老对称标准，56 位太短已淘汰；「不是群」才让 3DES 真正有意义。
- **SHA-256 / MD5**：哈希函数；MD5 已被攻破（能人造碰撞），安全场景弃用。
- **KDF**：把弱密码派生成强密钥（加盐 + 故意放慢，对抗暴力破解）。

**最容易混的三件事**
1. 加密 ≠ 哈希（加密可逆，哈希不可逆）。
2. DH / DSA 不加密内容（一个换钥匙、一个签名）。
3. MD5、DES 是「反面教材」，别在安全场景使用。

## 4. 非对称加密的群论原理

**为什么必须用群**：加密必须是可逆运算，而群保证每个元素都有逆元、运算良定义——这就是解密的数学兜底。

**通用配方（RSA / DH / ECC 本质都一样）**
1. 选一个有限群 `G` 和它的生成元 `g`；
2. **私钥** = 一个秘密整数 `a`（藏进指数）；
3. **公钥** = `gᵃ`（在群里把 g 自己乘自己 a 次）；
4. **安全性** = 离散对数问题 (DLP)：已知 `g` 和 `gᵃ`，求 `a` 极难。

**三种具体群**
- **RSA —— 乘法群 (Z/nZ)^×**：集合是与 n 互质的剩余类，运算是乘法 mod n，群的阶 = `φ(n) = (p−1)(q−1)`。欧拉定理 `m^φ ≡ 1` 保证 `m^(ed) = m`，`d` 是 `e` 在 mod φ(n) 下的逆元。陷阱门：算 `d` 需要知道 `φ(n)`，而 `φ(n)` 只有分解 `n=pq` 才能算——所以 RSA 安全来自整数分解，结构是有限乘法群。
- **DH / ElGamal / DSA —— 循环子群 ⟨g⟩**：群是 `(Z/pZ)^×` 里由 `g` 生成的素数阶子群。DH 双方各选 `a, b`，交换 `gᵃ, gᵇ`，都能算出 `g^(ab)`；第三者只有 `g, gᵃ, gᵇ`，要从 `gᵃ` 反推 `a` 就是 DLP（难）。
- **ECC —— 椭圆曲线点群 E(F_q)**：群元素不是数字而是「曲线上的点的坐标」，运算是点加；标量乘法 `k·P = P+…+P` 对应 RSA/DH 里的「幂」。公钥 `k·P`、私钥 `k`，反求 `k` 是 ECDLP（更难 → 同等安全下密钥更短）。

**DH 小例子**
- 公开参数：素数 `p = 23`，生成元 `g = 5`（5 是 mod 23 的原根）
- Alice 私钥 `a = 6` → 公钥 `A = 5⁶ mod 23 = 8`
- Bob 私钥 `b = 15` → 公钥 `B = 5¹⁵ mod 23 = 19`
- 交换 `A=8, B=19`
- Alice：`s = 19⁶ mod 23 = 2`；Bob：`s = 8¹⁵ mod 23 = 2`
- 双方得到 `s = 2`，窃听者只有 `5, 8, 19`，反推 `a` 是 DLP（难）

> 小知识：ECC 的点群是**阿贝尔（可交换）群**；「群基密码学」的量子方向研究**非交换群**（如辫群），因为非交换群上的某些问题连量子计算机也难解——即后量子专著的出发点。

**一句话收束**：RSA / DH / ECC 底层都是「在有限群里把私钥藏成指数，公钥即 `g^指数`；可逆性由群保证，安全性由反求指数很难保证」。

## 5. 公式与群论对照表（符号 + 来源）

### 符号说明 (Symbols)

| 符号 | 含义 |
|------|------|
| $\mathbb{Z}$ | 整数集 (integers) |
| $\mathbb{Z}/n\mathbb{Z}$ | 模 $n$ 的剩余类环 (residue classes mod $n$) |
| $(\mathbb{Z}/n\mathbb{Z})^\times$ | 与 $n$ 互质的剩余类，构成模 $n$ **乘法群** (units mod $n$) |
| $\mathbb{F}_q$ | 含 $q$ 个元素的**有限域** (finite field of order $q$) |
| $\langle g \rangle$ | 由 $g$ 生成的（循环）子群 (subgroup generated by $g$) |
| $\operatorname{ord}(g)$ | 元素 $g$ 的阶：最小正整数 $k$ 使 $g^k = e$ |
| $\varphi(n)$ | 欧拉函数：小于 $n$ 且与 $n$ 互质的正整数个数 |
| $\equiv$ | 同余 (congruent) |
| $\bmod$ | 模运算 (modulo) |
| $\oplus$ | 异或 XOR |
| $|G|$ | 群 $G$ 的阶（元素个数）(order / size of $G$) |
| $\approx$ | 约等于 (approximately) |
| $\subseteq\,/\,\in$ | 子集 / 属于 (subset / element of) |
| $E(\mathbb{F}_q)$ | 定义在有限域 $\mathbb{F}_q$ 上的**椭圆曲线点群** |
| $L_n[1/3,c]$ | 亚指数复杂度记号（介于多项式与指数之间） |

### 简化对照表（含算法来源）

| 算法 | 来源 / Origin | 核心公式 | 群结构 | 困难问题 | 安全强度 |
|------|--------------|----------|--------|----------|----------|
| **RSA** | Rivest, Shamir, Adleman，1977 | $c=m^e \bmod n;\; m=c^d \bmod n;\; ed\equiv1\pmod{\varphi(n)}$ | $(\mathbb{Z}/n\mathbb{Z})^\times$ 乘法群，阶 $\varphi(n)=(p-1)(q-1)$ | 大整数分解 (Factoring) | 亚指数 $L_n[1/3]$ (GNFS) |
| **DH** | Diffie & Hellman，1976 | $A=g^a;\; B=g^b;\; K=g^{ab}$ | $\langle g\rangle\subseteq(\mathbb{Z}/p\mathbb{Z})^\times$，素数阶 $q$ | DLP / CDH / DDH | 亚指数 (GNFS) |
| **ElGamal** | Taher ElGamal，1985 | $c_1=g^y;\; c_2=m\cdot h^y;\; m=c_2\cdot c_1^{-x};\; h=g^x$ | $\langle g\rangle$（同 DH） | DDH / DLP | 亚指数（同 DH） |
| **DSA** | NIST，1991（FIPS 186, 1994） | $r=(g^k\bmod p)\bmod q;\; s=k^{-1}(H(m)+xr)\bmod q$ | $\langle g\rangle,\ \operatorname{ord}(g)=q$（同 DH） | DLP | 亚指数（同 DH） |
| **ECC** | Koblitz & Miller，1985（独立提出） | $E:y^2=x^3+ax+b;\; P+Q=R$ | $E(\mathbb{F}_q)$ 椭圆曲线点群 (Abel)，阶 $\approx q+1\pm2\sqrt{q}$ | ECDLP | 指数 $O(2^{n/2})$ (Pollard $\rho$) |
| **AES** | Daemen & Rijmen（Rijndael），NIST 2001 | $\mathbb{F}_{2^8}\cong\mathbb{F}_2[x]/(x^8+x^4+x^3+x+1)$；SubBytes+ShiftRows+MixColumns+AddRoundKey | 加法 $(\mathbb{F}_{2^8},+)$ Abel 2-群；乘法 $(\mathbb{F}_{2^8}^\times,\cdot)$ 255 阶循环群 | 无已知数学简化 (SPN) | 指数 $2^{128}/2^{192}/2^{256}$ |
| **DES** | IBM（Lucifer）→ NIST，1977 | Feistel: $L_i=R_{i-1};\; R_i=L_{i-1}\oplus F(R_{i-1},K_i)$，16 轮 | 置换集合 $\{E_K\}$，**已证明非群** | 穷举密钥 | $2^{56}$（已破解） |
| **MD5** | Ronald Rivest，1991（RFC 1321） | $H_i=f(H_{i-1},M_i)$，Merkle-Damgård | 非群（压缩函数迭代） | 碰撞抵抗 | $2^{39}$（已破解） |
| **SHA-1** | NIST，1993（FIPS 180） | 同 MD5 结构 | 非群（压缩函数迭代） | 碰撞抵抗 | $2^{63}$（已破解） |
| **SHA-256** | NIST，2001（FIPS 180-2，SHA-2） | 同 MD5 结构 | 非群（压缩函数迭代） | 碰撞抵抗 | $2^{128}$（安全） |

> 时间复杂度多为 $O((\log N)^3)$（模幂 / 标量乘）或 $O(1)$（分组密码每块）；**亚指数 < 指数**，指数级才是理想安全。

## 6. 文献分类与延伸阅读

### 文献按性质分 5 类

| 类别 | 在讲什么 |
|------|----------|
| 1. 综述与导引 | 给进入密码学的数学人「指路」：群论如何进入现代密码学，附文献路径 |
| 2. 群论 / 非交换密码学专著 | 系统性专著：安全性来自群中算法问题（含非阿贝尔群），含量子时代考量 |
| 3. 权威手册与通用教材 | 经典算法在有限群 / 有限域上的权威推导，是「标准参考书」 |
| 4. 各算法专门文献 | 按 RSA / DH-ElGamal / DSA / ECC / AES / DES 拆开，讲清各自的群结构与标准 |
| 5. 入门科普 | 用通俗语言从 RSA 讲到 ECC，先建立整体直觉 |

**阅读顺序建议**：入门科普 → 综述导引 → 手册教材 → 分算法文献 → 群论专著（后量子前沿）。

## 7. 附录：可运行代码 rsa_demo.py

```python
# RSA 小例子 + 数字签名演示（极小数字，便于理解）
def modinv(e, phi):
    for d in range(2, phi):
        if (e * d) % phi == 1:
            return d

# 密钥生成
p, q = 3, 11
n = p * q                # 33
phi = (p - 1) * (q - 1)  # 20
e = 3
d = modinv(e, phi)       # 7

print("公钥 (n, e) =", (n, e))
print("私钥 (n, d) =", (n, d))

# 加密 / 解密
m = 4
c = pow(m, e, n)         # 加密 -> 31
m_back = pow(c, d, n)    # 解密 -> 4
print("\n[加密] 原文", m, "-> 密文", c)
print("[解密] 密文", c, "-> 原文", m_back)

# 数字签名
doc = 4
sig = pow(doc, d, n)     # 签名 -> 16
verify = pow(sig, e, n)  # 验证 -> 4
print("\n[签名] 原文", doc, "-> 签名值", sig)
print("[验证] 签名", sig, "->", verify, "（等于原文 => 签名有效）")
```
