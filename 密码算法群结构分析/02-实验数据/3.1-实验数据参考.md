## 实验一：MD4 碰撞

### 实验设置

| 属性 | 值 |
|---|---|
| **算法** | MD4 |
| **输出长度** | 128-bit |
| **轮数** | 3 轮，每轮 16 步 |
| **攻击方法** | 差分攻击 |
| **文献** | Wang et al. (2005) |

### 实验结果

| 属性 | 值 |
|---|---|
| **碰撞状态** | 成立 |
| **碰撞复杂度** | 秒级（极低） |
| **输入** | 两个视觉上完全不同的输入 |
| **输出** | MD4 哈希值完全相同 |

### 对照试验

| 哈希函数 | 碰撞状态 |
|---|---|
| MD4 | 碰撞成立 |
| SHA-256 | 碰撞不成立（哈希值不同） |

### 分析要点

1. **MD4 碰撞的根源**: 差分路径控制——攻击者可以"引导"消息差异在压缩过程中相互抵消
2. **MD4 弱点**: 只有 3 轮，非线性函数简单——差分路径容易构造
3. **SHA-256 抵抗原因**: 
   - 消息扩展（16 blocks → 64 words）放大微小差异
   - 非线性函数（$\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$）混合更充分

## 实验二：SHA-1 碰撞

### 实验设置

| 属性 | 值 |
|---|---|
| **算法** | SHA-1 |
| **输出长度** | 160-bit |
| **轮数** | 80 轮 |
| **攻击方法** | SHAttered |
| **文献** | Google + CWI (2017) |

### 实验结果

| 属性 | 值 |
|---|---|
| **碰撞状态** | 成立 |
| **碰撞复杂度** | ~$2^{63}$ 次 SHA-1 计算 |
| **等效计算** | ~110 GPU-年 |
| **输入** | 两个不同 PDF |
| **输出** | SHA-1 值完全相同 |

### 与 MD4 对比

| 属性 | MD4 | SHA-1 |
|---|---|---|
| 输出长度 | 128-bit | 160-bit |
| 轮数 | 3 轮 | 80 轮 |
| 碰撞复杂度 | 秒级 | ~$2^{63}$ |
| 攻击难度 | 简单 | 较复杂但仍可行 |

### 对 Git 的影响

- Git 使用 SHA-1 标识提交
- 理论上有风险
- 实践中被利用极难（因为还需要匹配 Git 的对象格式）

## 实验对比总表

| 哈希函数 | 输出长度 | 经典碰撞复杂度 | 量子碰撞复杂度 | 安全状态 |
|---|---|---|---|---|
| MD4 | 128-bit | $O(2^{10})$（秒级） | — | 已破 |
| SHA-1 | 160-bit | $O(2^{63})$（~110 GPU-年） | $O(2^{32})$（Grover） | 已破 |
| SHA-256 | 256-bit | $O(2^{128})$ | $O(2^{85})$ | 安全 |

## 时间复杂度对比

| 哈希函数 | 每块时间复杂度 |
|---|---|
| MD5 | $O(1)$（64 步） |
| SHA-1 | $O(1)$（80 步） |
| SHA-256 | $O(1)$（64 步） |

注：所有哈希函数都是 $O(n)$（线性于输入长度），每块操作次数固定。

## 写作提示

### 3.1.1 MD4 碰撞

1. **设置**: MD4（128-bit，3 轮），Wang et al. 差分攻击
2. **结果**: 碰撞成立，两个输入 MD4 哈希值相同
3. **分析**: 差分路径控制、轮数少、非线性函数简单
4. **过渡**: "MD4 的脆弱性让密码学家设计了更强的 MD5。但 MD5 也被破解了。"

### 3.1.2 SHA-1 碰撞

1. **设置**: SHA-1（160-bit，80 轮），SHAttered 攻击
2. **结果**: 碰撞存在，两个 PDF SHA-1 值相同
3. **分析**: SHA-1 比 MD4 难但终究被碰、对 Git 的影响
4. **过渡**: "哈希碰撞展示了算法设计缺陷的后果。"

## 注意事项

1. **不展示实际碰撞消息对**: 太长，仅陈述"碰撞已产生"并引用公开结果
2. **强调 MD4 太弱**: 只需要几秒钟就能碰撞
3. **强调 SHA-1 风险**: 虽然碰撞成本高于 MD4，但对于有资源的攻击者是可行的
4. **过渡到 §3.2**: "哈希碰撞展示了算法设计缺陷的后果。类似的原理也适用于非对称密码。"
