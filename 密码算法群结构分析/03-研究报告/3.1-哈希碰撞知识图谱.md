## 核心主题

**哈希碰撞演示**：展示弱哈希函数（MD4、SHA-1）在实际攻击下如何产生碰撞，对比强哈希（SHA-256）的安全性。

---

## 知识节点

### 1. MD4 碰撞（Wang et al. 2005）

| 属性 | 值 |
|---|---|
| **算法** | MD4（Rivest, 1990） |
| **输出长度** | 128-bit |
| **轮数** | 3 轮，每轮 16 步 |
| **攻击方法** | 差分攻击（Wang et al., 2005） |
| **攻击原理** | 差分路径控制——引导消息差异在压缩过程中相互抵消 |
| **碰撞复杂度** | 极低（几秒钟即可碰撞） |
| **安全弱点** | 轮数少（仅 3 轮）、非线性函数简单、差分路径容易构造 |
| **文献** | Wang, X., Yin, P., & Zhang, H. (2005). "Selected Differential-Multivariate Cryptanalysis of MD4". |

**群论视角**：
- MD4 不基于群结构，而是 Merkle-Damgård 迭代构造
- 压缩函数使用 $(\mathbb{Z}/2^{32}\mathbb{Z}, +)$ 模加和位运算
- 安全来自混淆与扩散，而非群困难问题

**实验要点**：
- 两个视觉上完全不同的输入，MD4 哈希值完全相同
- 对照试验：同样的输入用 SHA-256 计算——碰撞不成立

---

### 2. SHA-1 碰撞（SHAttered 2017）

| 属性 | 值 |
|---|---|
| **算法** | SHA-1（NIST, 1995） |
| **输出长度** | 160-bit |
| **轮数** | 80 轮 |
| **攻击方法** | SHAttered（Google + CWI, 2017） |
| **攻击复杂度** | ~$2^{63}$ 次 SHA-1 计算（~110 GPU-年） |
| **碰撞结果** | 两个不同 PDF 的 SHA-1 值相同 |
| **安全弱点** | 虽然比 MD4 更难，但仍被实际攻破 |
| **文献** | "SHAttered: Practical Attacks on SHA-1" (Google Project Zero, 2017) |

**群论视角**：
- SHA-1 同样基于 Merkle-Damgård 构造
- 压缩函数使用模加和位运算
- 与 MD4 相比：更多轮数（80 vs 3）、更大状态（160-bit vs 128-bit）

**对 Git 的影响**：
- Git 使用 SHA-1 标识提交
- 理论上有风险，但实践中被利用极难（需要匹配 Git 对象格式）

---

### 3. SHA-256（当前安全标准）

| 属性 | 值 |
|---|---|
| **算法** | SHA-256 |
| **输出长度** | 256-bit |
| **轮数** | 64 轮（每轮操作更复杂） |
| **碰撞复杂度** | ~$2^{128}$（生日攻击） |
| **安全状态** | 当前安全下限 |
| **攻击方法** | 生日攻击（量子计算机下 Grover 降至 ~$2^{85}$） |

**抵抗原因**：
- 消息扩展（16 blocks → 64 words）放大微小差异
- 非线性函数（$\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$）混合更充分
- 轮数更多、设计更复杂

## 实验对比

| 哈希函数 | 输出长度 | 经典碰撞复杂度 | 量子碰撞复杂度 | 安全状态 |
|---|---|---|---|---|
| MD4 | 128-bit | $O(2^{10})$（秒级） | — | 已破 |
| SHA-1 | 160-bit | $O(2^{63})$（~110 GPU-年） | $O(2^{32})$（Grover） | 已破 |
| SHA-256 | 256-bit | $O(2^{128})$ | $O(2^{85})$ | 安全 |

## 关键概念

### 1. 哈希碰撞定义

哈希碰撞是指两个不同的输入产生相同的哈希值：
$$H(m_1) = H(m_2) \quad \text{where} \quad m_1 \neq m_2$$

### 2. 三种安全属性

| 属性 | 定义 | 攻击复杂度 |
|---|---|---|
| **原像抵抗** | 给定 $h$，求 $m$ 使 $H(m) = h$ | $O(2^n)$（$n$ 为输出长度） |
| **第二原像抵抗** | 给定 $m$ 和 $h = H(m)$，求 $m' \neq m$ 使 $H(m') = h$ | $O(2^n)$ |
| **碰撞抵抗** | 求任意 $m_1, m_2$ 使 $H(m_1) = H(m_2)$ | $O(2^{n/2})$（生日攻击） |

### 3. Merkle-Damgård 构造

哈希函数的标准构造方式：
1. 初始向量（IV）
2. 分块输入消息
3. 迭代压缩函数 $f$
4. 输出最终哈希值

$$H_0 = \text{IV}, \quad H_i = f(H_{i-1}, M_i), \quad H_n = \text{Final Hash}$$

## 写作提示

1. **不展示实际碰撞消息对**：太长，仅陈述"碰撞已产生"并引用公开结果
2. **强调 MD4 太弱**：只需要几秒钟就能碰撞
3. **强调 SHA-1 风险**：虽然碰撞成本高于 MD4，但对于有资源的攻击者是可行的
