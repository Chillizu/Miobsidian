## 知识节点索引

### MD4 碰撞节点

| 属性 | 值 |
|---|---|
| 算法 | MD4 (Rivest, 1990) |
| 输出 | 128-bit |
| 轮数 | 3 轮，每轮 16 步 |
| 攻击 | 差分攻击 (Wang et al., 2005) |
| 复杂度 | 秒级 |
| 弱点 | 轮数少、非线性函数简单 |

### SHA-1 碰撞节点

| 属性 | 值 |
|---|---|
| 算法 | SHA-1 (NIST, 1995) |
| 输出 | 160-bit |
| 轮数 | 80 轮 |
| 攻击 | SHAttered (Google + CWI, 2017) |
| 复杂度 | ~$2^{63}$ (~110 GPU-年) |
| 弱点 | 虽比 MD4 难但仍被攻破 |

### SHA-256 安全节点

| 属性 | 值 |
|---|---|
| 算法 | SHA-256 |
| 输出 | 256-bit |
| 轮数 | 64 轮 |
| 安全 | ~$2^{128}$ |
| 状态 | 当前安全标准 |

## 写作要点索引

### 3.1.1 MD4 碰撞

| 要点 | 内容 |
|---|---|
| 设置 | MD4（128-bit，3 轮），Wang et al. 差分攻击 |
| 结果 | 碰撞成立，两个输入 MD4 哈希值相同 |
| 分析 | 差分路径控制、轮数少、非线性函数简单 |
| 过渡 | "MD4 的脆弱性让密码学家设计了更强的 MD5。" |

### 3.1.2 SHA-1 碰撞

| 要点 | 内容 |
|---|---|
| 设置 | SHA-1（160-bit，80 轮），SHAttered 攻击 |
| 结果 | 碰撞存在，两个 PDF SHA-1 值相同 |
| 分析 | SHA-1 比 MD4 难但终究被碰、对 Git 的影响 |
| 过渡 | "哈希碰撞展示了算法设计缺陷的后果。" |
