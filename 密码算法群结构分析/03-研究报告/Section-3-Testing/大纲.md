## 3.1 哈希碰撞演示（~300 words）

### 本节要传达的核心信息

哈希碰撞不是理论上的可能性——对弱哈希（MD4、SHA-1），它是实际可操作的攻击。强哈希（SHA-256）在现实条件下不可碰撞。

---

### 3.1.1 MD4 碰撞

**实验目标**：展示弱的哈希函数在实际中如何产生碰撞——两段不同的输入产生完全相同的哈希值。

**讲解顺序**：

1. **设置**：
   - MD4（Rivest, 1990）——128-bit 输出，3 轮，每轮 16 步
   - 使用 Wang et al.（2005）的差分攻击方法：分析消息差异如何在压缩函数中传播
   - 不展示具体的碰撞消息对（太长），但陈述这些碰撞对已存在

2. **结果**：
   - MD4：碰撞成立——两个视觉上完全不同的输入，MD4 哈希值完全相同
   - 对照试验：同样的输入用 SHA-256 计算——碰撞不成立，哈希值不同

3. **分析**：
   - MD4 碰撞的根源：**差分路径控制**——攻击者可以"引导"消息差异在压缩过程中相互抵消
   - MD4 只有 3 轮，非线性函数简单——差分路径容易构造
   - SHA-256 抵抗的原因：
     - 消息扩展（16 blocks → 64 words）放大微小差异
     - 非线性函数（$\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$）混合更充分
   - 教训：轮数和非线性函数的设计不是"越多越好"——而是需要精心构造才能抵抗差分分析

**这段写完后读者应该理解**：MD4 太弱——只需要几秒钟就能碰撞。下次看到 MD4 签名时可以确信它不安全。

**过渡到 3.1.2**："MD4 的脆弱性让密码学家设计了更强的 MD5。但 MD5 也被破解了。接下来看 SHA-1——曾经是安全标准，现在也已被攻破。"

---

### 3.1.2 SHA-1 碰撞

**实验目标**：展示 SHA-1 的削弱——曾是安全标准，现已被实际碰撞攻击攻破。

**讲解顺序**：

1. **设置**：
   - SHA-1（NIST, 1995）——160-bit 输出，80 轮
   - SHAttered（2017，Google + CWI）：生成两个不同 PDF，SHA-1 值相同
   - 计算复杂度：~$2^{63}$ 次 SHA-1 计算（等价于 ~110 GPU-年）

2. **结果**：
   - 碰撞存在——两个 PDF（不同内容）的 SHA-1 完全相同
   - 这次攻击不是理论上的——Google 实际生成了碰撞并公开了这对 PDF（可引用但不展示内容）

3. **分析**：
   - SHA-1 比 MD4 更难碰的原因：更多轮数（80 vs 3）、更大的状态（160-bit vs 128-bit）
   - 但终究被碰了——这意味着什么？
   - 对 Git（使用 SHA-1 标识提交）的安全影响：理论上有风险，实践中被利用极难（因为还需要匹配 Git 的对象格式）
   - SHA-256 目前是安全下限

**这段写完后读者应该理解**：SHA-1 在今天不应再被视为安全的——虽然碰撞它的成本远高于 MD4，但对于有资源的攻击者是可行的。

**过渡到 3.2**："哈希碰撞展示了算法设计缺陷的后果。类似的原理也适用于非对称密码——错误的参数选择会让安全算法变得不安全。"

---

## 3.2 参数不当攻击（~300 words）

### 本节要传达的核心信息

安全算法的安全性依赖于正确的参数选择。选择了太小的素数（DLP）或异常的曲线（ECC），算法就失效了。

---

### 3.2.1 小素数 DLP

**实验目标**：展示 DLP 在不当参数下不再困难。

**讲解顺序**：

1. **设置**：
   - 使用小的素数 $p$（如 $p = 2^{32} - 5$，约 32-bit）
   - 选择生成元 $g$
   - 公开 $g^a$，目标：求 $a$

2. **执行**：
   - 使用 Pohlig–Hellman 算法：如果 $p-1$ 是平滑数（小素因子之积），DLP 可分解为多个小阶子群中的 DLP
   - 或使用 baby-step giant-step：$O(\sqrt{p})$ 时间

3. **结果**：
   - DLP 在 32-bit 素数的群中可在几秒内求解
   - 对比：2048-bit 素数需要 $2^{112}$ 量级的操作（当前不可行）

4. **分析**：
   - DLP 的难度取决于群的大小（$p$ 的 bit 长度）和群的结构（$p-1$ 是否有大素因子）
   - 即使使用大 $p$，如果 $p-1$ 是平滑数，Pohlig–Hellman 也会大大降低难度
   - 实践要求：$p$ 至少 2048-bit，$p-1$ 至少有一个大素因子

**这段写完后读者应该理解**：DLP 不是"数学上困难"——而是"在有适当参数时计算上困难"。"困难"依赖于参数选择。

**过渡到 3.2.2**："类似的问题在椭圆曲线密码中也存在——某些看似无害的曲线选择会导致 ECDLP 容易求解。"

---

### 3.2.2 弱 ECC 曲线

**实验目标**：展示如果椭圆曲线选择不当，ECDLP 不再困难。

**讲解顺序**：

1. **设置**：
   - 选择一条异常曲线（anomalous curve）：$E(\mathbb{F}_p)$ 的阶等于 $p$，即 $\#E(\mathbb{F}_p) = p$
   - 这是非常特殊的情况——大多数曲线的阶都在 $p+1 \pm 2\sqrt{p}$ 范围内（Hasse 不等式）

2. **执行**：
   - Smart 攻击：将 $\mathbb{F}_p$ 上的椭圆曲线提升到 $\mathbb{Q}_p$（$p$-adic 数），然后将 ECDLP 映射到加法群 $\mathbb{F}_p^+$ 上的 DLP
   - 在加法群上 DLP 是平凡的（$kP$ 就是 $k$ 次加法的结果）

3. **结果**：
   - ECDLP 在异常曲线上可在 $O(\log p)$ 时间内求解——多项式时间！
   - 对比正常情况下 ECDLP 需要 $O(\sqrt{p})$（指数时间）

4. **分析**：
   - 这个攻击不适用于随机选择的曲线——因为随机曲线的阶等于 $p$ 的概率极低（约 $1/\sqrt{p}$）
   - 但展示了：椭圆曲线的安全性依赖于曲线参数的精心选择
   - 实际含义：NIST 曲线（P-256、P-384 等）经过严格验证；使用未知来源的曲线有风险

**这段写完后读者应该理解**：ECC 安全的前提是曲线参数正确。不能随便选一条曲线就认为安全——验证参数和标准曲线很重要。

## 与整篇论文的连接

1. **MD4/SHA-1 结果 → §2.2 安全表格的验证**：
   - §2.2 说 MD5 安全是 $2^{39}$，SHA-1 是 $2^{63}$
   - 实验中实际产生了碰撞——验证了这些理论边界

2. **小素数 DLP → §1.1.1 理论到实践**：
   - §1.1.1 说"哪些群上的 DLP 困难"——实验中展示了"不良群"上的 DLP 容易

3. **弱 ECC 曲线 → §1.2.1 ECC 理论到实践**：
   - 展示了曲线参数的选择至关重要

4. **所有实验 → Conclusion 量子威胁**：
   - 实验展示的都是"经典计算攻击"
   - 量子威胁下，即使是正确参数的 RSA/DH/ECC 也不安全
   - 自然引出后量子密码的必要性
