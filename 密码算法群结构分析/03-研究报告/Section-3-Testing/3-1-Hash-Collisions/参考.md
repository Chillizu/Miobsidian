# 3.1: Hash Collision in Weak MD4 and SHA-1 — 完整参考

> 本节涵盖两个子实验：MD4 碰撞（Wang 2005）和 SHA-1 碰撞（SHAttered 2017）。
> 核心命题：弱哈希的设计缺陷使实际碰撞成为可能，而强哈希（SHA-256）不受影响。

---

## 一、哈希函数基础

### Merkle–Damgård 构造

所有三种哈希（MD4、SHA-1、SHA-256）共用此迭代构造：

```
输入消息 M → 填充对齐 → 分割为 512-bit 块 M_i
H_0 = IV (固定初始向量)
for i = 1..k:
    H_i = f(H_{i-1}, M_{i-1})
输出 H_k
```

### 碰撞的定义

$H(x) = H(y)$ 且 $x \neq y$。密码学安全的哈希应使碰撞在计算上不可行。

### 碰撞搜索复杂度

|| 理想安全 (生日边界) | 实际攻击 |
|:---|---:|---:|
| 128-bit 哈希 (MD4) | $2^{64}$ | $2^{39}$ |
| 160-bit 哈希 (SHA-1) | $2^{80}$ | $2^{63}$ |
| 256-bit 哈希 (SHA-256) | $2^{128}$ | 无已知攻击 |

## 二、实验 a)1): MD4 碰撞

### MD4 规格

| 参数 | 值 |
|:---|:---|
| 设计者 | Rivest (1990) |
| 输出 | 128-bit |
| 轮数 | 3 轮，每轮 16 步 = 48 步 |
| 每轮布尔函数 | $F(x,y,z) = (x \land y) \lor (\lnot x \land z)$ |
| | $G(x,y,z) = (x \land y) \lor (x \land z) \lor (y \land z)$ |
| | $H(x,y,z) = x \oplus y \oplus z$ |
| 初始向量 | `0x67452301 efcdab89 98badcfe 10325476` |

### 攻击方法 (Wang et al. 2005)

**论文**：Xiaoyun Wang, Xuejia Lai, Dengguo Feng, Hui Chen, Xiuyuan Yu.
"Cryptanalysis of the Hash Functions MD4 and RIPEMD".
EUROCRYPT 2005, LNCS 3494. [PDF](https://iacr.org/archive/eurocrypt2005/34940001/34940001.pdf)

**核心思路——精确差分攻击**：

1. **消息差异**：
   - $\Delta m_1 = 2^{31}$
   - $\Delta m_2 = 2^{31} - 2^{28}$
   - $\Delta m_{12} = -2^{16}$
   - 其他 $m_i$ 无差异

2. **差分路径**：精确描述每步中状态差异如何传播（Table 5 原文）。包含两个内部碰撞段：step 2–25 和 step 36–41。

3. **充分条件**：共 122 个比特条件（Table 6 原文），例如 step 9 需要 10 个条件确保 $b_2$ 和 $c_2$ 的差异不影响 $a_3$。

4. **消息修改技术**：
   - 单步修改：直接调整消息字以满足第一轮条件 → 成功率 $2^{-25}$
   - 多步修改：利用第一轮中的局部碰撞来修正第二轮条件 → 成功率 $2^{-2} \sim 2^{-6}$

**复杂度**：$2^{39}$ 次 MD4 压缩 ≈ 现代 CPU 几秒到几分钟。

### 对照实验 (SHA-256)

相同差异的消息 → SHA-256 不产生碰撞。
原因：消息扩展 16→64 words 放大微小差异，6 种非线性函数混合更充分。

### 关键教训

- 3 轮 **不够**——48 步压缩不足以扩散差异
- 简单非线性函数（$F, G, H$）使差分路径容易构造
- "轮数越多 = 越安全"的直觉正确但不充分——轮间设计和消息扩展同样重要

## 三、实验 a)2): SHA-1 碰撞

### SHA-1 规格

| 参数 | 值 |
|:---|:---|
| 设计者 | NIST (1995) |
| 输出 | 160-bit |
| 轮数 | 80 轮 (4 段 × 20 轮) |
| 每段布尔函数 | $f_{IF}, f_{XOR}, f_{MAJ}, f_{XOR}$ |
| 消息扩展 | 16 words → 80 words |
| 初始向量 | `67452301 efcdab89 98badcfe 10325476 c3d2e1f0` |

### 攻击方法 (SHAttered 2017)

**论文**：Marc Stevens, Elie Bursztein, Pierre Karpman, Ange Albertini, Yarik Markov.
"The first collision for full SHA-1". CRYPTO 2017. [PDF](https://shattered.io/static/shattered.pdf)

**攻击流程**：

1. **扰动向量 (DV) 选择**：II(52,0)，确定每个局部碰撞的位置
2. **非线性差分路径 (NL path)**：前 16 步连接输入链接值差异和线性部分
3. **联合局部碰撞分析 (JLCA)**：穷举评估所有可行的差分路径，选取最优成功概率
4. **分布式 GPU 搜索**：
   - 第一近碰撞块：使用修改后的 Stevens 2013 代码
   - 第二近碰撞块：使用 Karpman 2015 GPU 框架 + 新增技术（boomerang、solvability fix）
5. **两个近碰撞块对 → 完整碰撞**

**计算投入**：
- $2^{63.1}$ SHA-1 压缩调用
- ≈ 6500 CPU-years + 100 GPU-years
- AWS 租金：~US$ 75K–110K

### 影响分析

| 场景 | 风险 | 实际可操作性 |
|:---|:---:|:---|
| TLS 证书签名 | 高 | 已禁止 (CABForum 2017) |
| Git commit id | 中 | 极低——需匹配 Git 对象格式 |
| PDF 签名 | 高 | SHAttered 已验证可操作 |
| 软件签名 | 中 | 已开始迁移 |

### SHA-256 为何仍然安全

| 特性 | SHA-1 | SHA-256 |
|:---|:---:|:---:|
| 输出 | 160-bit | 256-bit |
| 轮数 | 80 | 64 |
| 消息扩展 | 16→80 words | 16→64 words |
| 非线性函数 | 4 种 | 6 种 ($\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$) |
| 最佳攻击 | $2^{63}$ (已破) | 无 (生日边界 $2^{128}$) |

## 四、时间线

```
1990  MD4 发布 (Rivest)
1991  MD5 发布 (Rivest) — MD4 的增强版
1995  SHA-1 发布 (NIST)
1996  Dobbertin 碰撞 MD4 (2^20)
2004  Wang 碰撞 MD5 (Crypto 2004 Rump Session)
2005  Wang 完整 MD4 碰撞 (Eurocrypt 2005, Best Paper)
2005  Wang 理论 SHA-1 攻击 (2^69)
2017  SHAttered: 首次实际 SHA-1 碰撞 (2^63)
现在  SHA-256 保持安全
```
