# 3.2: Use Inappropriate Parameters to Test Asymmetric Ciphers — 完整参考

> 本节涵盖两个子实验：小素数 DLP 和弱 ECC 曲线（异常曲线）。
> 核心命题：安全算法的安全性**不等于**算法本身安全——它依赖于**正确的参数选择**。

---

## 一、核心概念

### DLP 的参数依赖

DLP（离散对数问题）的难度取决于两个因素：

```
DLP 难度 = f(群的大小, 群的结构)
             ↑            ↑
           p 的 bit 长度  p-1 是否平滑
```

- **群的大小**：$|G| \approx p$，大 $p$ → 难
- **群的结构**：$p-1$ 分解为小素因子之积 → Pohlig–Hellman 可降阶攻击

### ECC 的参数依赖

ECDLP 的难度取决于：

```
ECDLP 难度 = f(曲线参数, 有限域特征)
```

- **标准曲线**：$\#E(\mathbb{F}_q) \approx q$（Hasse 不等式中心），ECDLP 指数时间
- **异常曲线**：$\#E(\mathbb{F}_q) = q$（Hasse 不等式边界），ECDLP 多项式时间

## 二、实验 b)1): 小素数 DLP

### 实验设计

| 参数 | 不安全配置 | 安全配置 |
|:---|:---:|:---:|
| $p$ | $2^{32} - 5$ (~32-bit) | $\geq 2^{2048}$ (~2048-bit) |
| 生成元 $g$ | 任意 | 素数阶子群生成元 |
| $p-1$ 因式分解 | 平滑 (小素因子) | 至少有一个大素因子 |
| 求解时间 | 秒级 | $\geq 2^{112}$ 次操作 |

### 攻击方法 1: Baby-step giant-step

**原理**：利用"时间-空间权衡"求解 DLP。

```
输入: 群 G 的生成元 g, 目标 h = g^a
设 m = ⌈√|G|⌉

baby steps: 预计算并存储 (g^j, j) 对 j = 0..m-1
giant steps: 对 i = 0..m-1:
              计算 h · (g^(-im))
              在 baby step 表中查找匹配
              找到则 a = i·m + j

复杂度: O(√p) 时间 + O(√p) 空间
```

**适用性**：$p = 2^{32}$ → $2^{16}$ 步 → 内存几 MB → 实时可解。
$p = 2^{2048}$ → $2^{1024}$ 步 → 不可行。

### 攻击方法 2: Pohlig–Hellman

**原理**：如果 $|G| = \prod p_i^{e_i}$，将 DLP 分解为多个小阶子群上的 DLP，再用 CRT 组合结果。

```
输入: 群 G 的生成元 g, 目标 h = g^a, 阶 n = |G| = ∏ p_i^{e_i}

对每个素因子 p_i:
    计算 g_i = g^(n/p_i^{e_i}) (阶为 p_i^{e_i})
    计算 h_i = h^(n/p_i^{e_i})
    在子群 ⟨g_i⟩ 中求解 DLP → a mod p_i^{e_i}
使用 CRT 组合: a = ∑ a_i · (n/p_i^{e_i}) · inv(n/p_i^{e_i} mod p_i^{e_i}) mod n

复杂度: O(∑ e_i · (log n + √p_i))
```

**关键**：如果 $p-1$ 的**最大素因子**较小，Pohlig–Hellman 显著降低难度。这就是为什么安全参数要求 $p-1$ 至少有一个大素因子。

### 实验结果

```
p = 2^32 - 5
  |G| = p - 1 = 2^32 - 6 = 2 · 3 · 5 · 17 · 257 · 65537 · 6700417
  → 最大素因子 ≈ 2^23 → Pohlig–Hellman 分解为 7 个子群 DLP
  → 总时间 << 1 秒

p = 2^2048 (安全)
  |G| ≈ 2^2048 → 最大素因子 ≈ 2^2047
  → Pohlig–Hellman 无效
  → baby-step GS 需 2^1024 步 → 不可行
```

## 三、实验 b)2): 弱 ECC 曲线 (Anomalous)

### 异常曲线

**定义**：椭圆曲线 $E(\mathbb{F}_p)$ 的阶等于 $p$，即 $\#E(\mathbb{F}_p) = p$。

**正常范围**（Hasse 不等式）：
$$|E(\mathbb{F}_q)| = q + 1 - t, \quad |t| \leq 2\sqrt{q}$$

正常曲线的阶在 $q+1 \pm 2\sqrt{q}$ 范围内。阶恰好等于 $q$（即 $t = 1$）的概率约 $1/\sqrt{q}$。

### 攻击方法: Smart's p-adic Lift

**论文**：Smart, N. P. "The Discrete Logarithm Problem on Elliptic Curves of Trace One".
Journal of Cryptology 12(3), 193–196, 1999.
[doi:10.1007/s001459900052](https://doi.org/10.1007/s001459900052)

**原理**：利用 $p$-adic 数 $\mathbb{Q}_p$ 上形式群的同态。

```
设 E 是 F_p 上的异常曲线 (|E(F_p)| = p)

Step 1: 将 E 提升到 Q_p 上的曲线 Ê
  使用 Hensel 引理将系数从 F_p 提升到 Z_p, 再扩展到 Q_p

Step 2: 构造形式群同态
  Ê(Q_p) 包含一个形式群 Ê₁(Q_p) ≅ (Z_p, +)
  这个形式群自然同态于 F_p 的加法群

Step 3: 映射 ECDLP → 加法群 DLP
  φ: E(F_p) → F_p^+
  φ(P) = (x(P̂)/y(P̂)) mod p  (其中 P̂ 是 P 的提升)

Step 4: 加法群中求解
  在 F_p^+ 中: DLP 平凡——kP = Q → k = φ(Q) · φ(P)^(-1) mod p
```

**复杂度**：$O(\log p)$（多项式时间）！

**对比**：正常 ECDLP 需 $O(\sqrt{p})$（指数时间）。

### 实验结果

```
p = 2^256 − 2^224 + 2^192 + 2^96 − 1 (P-256 的 p)
  
异常曲线: 假设 |E| = p
  Smart 攻击: O(log p) ≈ 256 步 → 毫秒级
  对比: 正常 ECDLP 在 P-256 上需 2^128 步 → 不可行

随机曲线: |E| 在 p+1±2√p 范围内
  概率 |E| = p 约为 1/√p ≈ 2^(-128) → 可忽略
  Smart 攻击不适用
```

### 防御

| 曲线 | 状态 | 原因 |
|:---|:---:|:---|
| NIST P-256 | 安全 | 已验证非异常 |
| NIST P-384 | 安全 | 同上 |
| Curve25519 | 安全 | Montgomery 曲线，经严格验证 |
| secp256k1 | 安全 | Bitcoin 使用，已验证 |
| 未知来源曲线 | **不推荐** | 可被植入异常曲线 |

## 四、实验设计总结

| 实验 | 错误参数 | 正确参数 | 攻击算法 | 复杂度 |
|:---|:---|:---|:---|---:|
| b)1) | $p = 32$-bit | $p \geq 2048$-bit | BSGS / PH | $O(\sqrt{p})$ 或 $O(\sum e_i\sqrt{p_i})$ |
