## 3.1 哈希碰撞演示 (~300 words)

### 3.1.1 MD4 碰撞

**实验目标**：展示弱的哈希函数如何在实际中产生碰撞——两段不同输入产生相同的哈希值。

**设置**：
- 使用公开的 MD4 碰撞生成工具（如 Wang 的方法实现）
- MD4 是 MD5 的前身，设计于 1990 年，3 轮 × 16 步 = 48 步压缩
- 对比 SHA-256（64 轮，更复杂的消息扩展）→ 展示"轮数更多、设计更仔细"的抗碰撞效果

**执行**：
- 选取两段不同的消息 $M_1$ 和 $M_2$ → 经 MD4 压缩产生相同哈希值 $H(M_1) = H(M_2)$
- 验证对 SHA-256 同样操作：$H_{\text{SHA-256}}(M_1) \neq H_{\text{SHA-256}}(M_2)$

**结果**：
- MD4：碰撞成立——两个视觉上完全不同的输入，哈希完全相同
- SHA-256：碰撞不成立——哈希值不同

**分析**：
- MD4 碰撞的根源：差分路径控制——攻击者可以"引导"消息差异在压缩过程中相互抵消
- SHA-256 抵抗此攻击的原因：消息扩展（$16$ blocks → $64$ words）放大微小差异；非线性函数（$\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$）混合更充分
- 教训：哈希函数的设计细节（轮数、非线性函数、消息扩展）直接决定安全性——不是"多加几轮"就能修复弱点

**写作提示**：不在此节展示实际的碰撞消息对（太长）。仅陈述"碰撞已产生，具体消息对见附录 A"或引用外部公开结果。

### 3.1.2 SHA-1 碰撞

**实验目标**：展示 SHA-1 的削弱——曾是安全标准，现已被实际碰撞攻击攻破。

**设置**：
- 引用 SHAttered（2017）的结果（Google + CWI Amsterdam）
- 两个不同的 PDF 文件 → 相同 SHA-1 哈希
- 计算资源：$2^{63}$ 次 SHA-1 压缩，约 $6500$ CPU-年 + $100$ GPU-年

**结果与分析**：
- SHA-1 的 $160$-bit 输出理论上需要 $2^{80}$ 次尝试（生日界）——攻击比理论低 $2^{17}$ 倍
- 原因：SHA-1 的压缩函数存在结构弱点——消息差分可在压缩轮中局部传播而不充分混淆
- 对比 SHA-256：$256$-bit 输出 + 更强的消息扩展 → 目前无已知碰撞攻击

**写作提示**：SHA-1 碰撞不要求读者亲自执行——引用已发表结果即可。重点是"$2^{63}$ 对比 $2^{80}$"这个数字——它量化了"攻击比理论脆弱多少"。

## 3.2 非对称加密参数不当测试 (~300 words)

### 3.2.1 小素数 DLP 破解

**实验目标**：展示——群参数的选择直接影响安全性。小的 $p$ → DLP 可手动求解；大的 $p$ → 不可行。

**设置**：
- DH/ElGamal 使用小素数 $p < 2^{20}$（如 $p = 100003$）
- 生成元 $g=2$（需验证 $2$ 是模 $p$ 原根）
- 给定 $g^a \bmod p$，用大步小步法求解 $a$

**执行（大步小步法）**：
- 设 $m = \lceil \sqrt{p} \rceil$。预计算 Baby Steps：存储 $(g^j, j)$ for $j = 0, \ldots, m-1$
- Giant Steps：计算 $h \cdot g^{-im}$ for $i = 0, \ldots, m-1$，查表匹配 → 碰撞给出 $a = im + j$
- 时间：$O(\sqrt{p})$ 次群运算——对 $p < 2^{20}$，约 $2^{10} = 1024$ 次运算，可秒级求解

**结果**：
- 小 $p$：$a$ 被秒级恢复 → 共享密钥 $g^{ab}$ 暴露
- 对比安全参数 $p \approx 2^{2048}$：$\sqrt{p} \approx 2^{1024}$——远远超出可计算范围

**分析**：
- 参数大小决定安全强度——不是群结构本身的问题，而是"群太小时 DLP 不够难"
- 实际标准（RFC 3526）强制使用 $2048$-bit 或更大的 $p$
- 量子威胁：即便 $p$ 足够大，Shor 算法仍可在多项式时间内求解——群大小不再提供保护

### 3.2.2 弱椭圆曲线参数

**实验目标**：展示——并非所有椭圆曲线都安全。选择不当的曲线可以使 ECDLP 变得脆弱。

**设置**：
- 选取退化曲线：(a) 奇异曲线（$\Delta = 0$，如 $y^2 = x^3$），(b) 异常曲线（$|E(\mathbb{F}_q)| = q$）
- 给定 $P$ 和 $Q = kP$，用不适合一般曲线的快速方法求解 $k$

**结果与分析**：
- 奇异曲线：ECDLP 可化简为乘法群 DLP 或加法群 DLP → 多项式时间求解
- 异常曲线（Smart 攻击）：将 $E(\mathbb{F}_q)$ 提升到 $p$-adic 域 → ECDLP 变线性方程 → 多项式时间 $O(\log^3 q)$ 求解
- 标准曲线（NIST P-256、secp256k1）均通过参数选择规避了这两类退化——选择曲线前需验证：(1) $\Delta \neq 0$（非奇异），(2) $|E(\mathbb{F}_q)| \neq q$（非异常），(3) 嵌入次数 $k$ 足够大（防 MOV 攻击）

**写作提示**：Smart 攻击的 $p$-adic 提升不展开详细推导——1 句概述即可。"多项式时间"足以传达"不安全"的结论。参考文献 [21] 提供完整技术细节。

## 写作注意事项

1. **不写伪代码。** 大步小步法的步骤用文字描述（"预计算 → 查表 → 匹配"）——这不等于代码。
2. **不要求读者跑实验。** 实验结果陈述为"可观察到"、"已验证"——引用公开碰撞结果作为证据。
3. **每个实验包含"教训"段落。** 哈希碰撞 → "设计细节决定安全"，参数不当 → "不是群的问题，是参数选择的问题"
4. **量子威胁作为实验的自然延伸。** 小素数 DLP 破解后：你可能会想"用大素数就行"→ 但这挡不住量子算法
