# §2 算法属性与密码分析 — 详细大纲

> 字数：~600 words。紧凑的三段——复杂度对比表、安全分析表、应用场景表。不做逐步推导——仅给结论和关键推理线索。

## 2.1 时间复杂度对比 (~200 words)

> 目标：让读者一眼看出"哪些算法慢、哪些快、为什么"。

### 内容

**表格** + 简短解释：

| 算法 | 核心运算 | 渐近复杂度 | 直觉解释 |
|:---|:---|:---:|:---|
| RSA | 平方-乘法模幂 | $O(\log^3 n)$ | 指数 $d$ 的位数 × 每次模乘的平方代价 |
| DH/ElGamal | 同上（2–3 次） | $O(\log^3 p)$ | 同量级，ElGamal 多 2–3 倍 |
| DSA | 模幂（短指数） | $O(\log q \cdot (\log p)^2)$ | 指数 256 位→迭代 1/8 的 RSA |
| ECC | 倍点-加法标量乘 | $O(\log^3 q)$ | 与 RSA 同量级，但 $\log q=256 \ll \log n=3072$ |
| AES/DES/Hash | 固定轮/步数 | $O(n)$ | 每块常数 → 总时间正比于消息长度 |

**解释重点（不要公式推导——只用一句话点出关键推理）：**

- 公钥算法的 $O(\log^3\cdot)$ 量级：位数翻倍 → 时间约增 $8$ 倍（因为 $\log$ 翻倍 → $2^3=8$）
- ECC 在同安全下比 RSA 快 $5$–$10$ 倍的原因：$256$-bit 的 $\log^3$ 远小于 $3072$-bit
- DSA 比 RSA 快的原因：指数仅 $256$ bits（模数仍 $2048$ bits）
- 对称/哈希算法的 $O(n)$：时间只与数据量有关，与密钥长度无关

**写作提示**：不在此处推导平方-乘法（那是上一稿 Ch3 的内容）。仅陈述结论 + 1 句直觉解释。如果想给"为什么"，用脚注引 §3 的实验。

### 2.2 安全分析与密码攻击 (~250 words)

> 目标：展示攻击模型→攻击代价→等效安全比特→量子威胁。核心信息以表格呈现。

| 算法（参数） | 安全归约 | 最优攻击 | 等效安全 | 量子威胁 |
|:---|:---|:---|:---:|:---:|
| RSA-2048 | 因子分解 | GNFS（亚指数） | ~112 bits | Shor 彻底攻破 |
| DH/DSA-2048 | DLP | GNFS-DLP（亚指数） | ~112 bits | Shor 彻底攻破 |
| ECC-256 | ECDLP | Pollard-$\rho$（指数） | 128 bits | Shor 彻底攻破 |
| AES-256 | — | Biclique / 穷举 | 256 bits | Grover 降至 128 bits |
| MD5 | — | 碰撞攻击 | $2^{39}$（已破） | — |
| SHA-1 | — | 碰撞攻击 | $2^{63}$（已破） | — |
| SHA-256 | — | 生日攻击 | 128 bits | Grover 降至 85 bits |

**解释重点：**

- "亚指数" vs "指数"的区别——一句话：RSA/DH 的安全性随着参数增大增长较慢（GNFS），ECC 的安全性随参数增大呈指数增长（Pollard-$\rho$）
- 阿贝尔群上的统一脆弱性：DHP/DLP/ECDLP 本质上都是"在阿贝尔群上找隐藏结构"→ Shor 算法通过量子 Fourier 变换找出群的周期 → 多项式时间求解
- 这就是 §1.1.2 中转向非阿贝尔群密码的根本动机——非阿贝尔群上的 CSP 不直接对应阿贝尔群的 HSP 结构，Shor 算法不适用

**写作提示**：不展开 GNFS $L_n[1/3, 1.923]$ 的具体公式——仅说"亚指数时间"并引用标准文献。Pollard-$\rho$ 同样只说"指数时间 $O(\sqrt{q})$"。复杂度的严格定义留给更高级的文献。

### 2.3 应用场景 (~150 words)

> 目标：让读者看到算法不只是"数学公式"——它们在实际世界中保护着数据。参考 postquantum.com。

| 算法 | 典型应用 |
|:---|:---|
| RSA | TLS 证书签名、传统 PKI、密钥传输 |
| DH | TLS 1.2 握手、VPN 密钥协商 |
| ECC (ECDH/ECDSA) | TLS 1.3、Bitcoin 交易签名（secp256k1）、iOS/Android 安全区 |
| DSA | 美国政府数字签名标准 |
| AES | 磁盘加密（BitLocker/FileVault）、TLS 对称会话加密、Wi-Fi WPA2 |
| MD5 | 仅校验和与缓存键——禁止用于数字签名 |
| SHA-256 | Bitcoin 工作量证明、TLS 证书签名、代码签名 |

**写作提示**：此表用 1 句引导文字即可——不需要展开描述每个应用的工作原理。"数字签名"、"密钥协商"这类术语假定读者已在 §1 中理解。增加一句关于"密码学透明性"的收尾：用户每天都在使用这些算法（打开手机、访问网页、发消息），感知不到其存在——这是密码学成功的标志。
