# Introduction — 写作指导

> 目标：~150 words。两段。用途是"抓住读者 + 给路线图"。
> 来源参考：[[密码算法群结构分析/03-研究报告/GTIC-注释翻译]] p1（密码学基本模型），[[密码算法群结构分析/03-研究报告/AOGTICA-注释翻译]] p1（群论与密码学总论）

---

## 第一段：数学 = 安全基础设施（~80 words）

**这一段要传达的核心论点**：没有群论等近现代数学工具，就没有现代密码学，也就没有我们今天依赖的网络安全。

**讲解顺序**：

1. 先陈述一个事实：群论（以及其他近现代数学概念）为密码学提供了安全基础设施。
   - 不需要讲群论定义，那是 §1 的事。
   - 这里只需要一个有力的开场陈述。

2. 然后展开：密码学家利用群上的计算困难问题（这里不需要具体命名 DLP 等，留到 §1），构建了从个人设备到互联网的各种加密方案。
   - 让读者感觉到"密码学无处不在"。

3. 最后上升到论点：数学本质上是信息安全的基础设施。
   - 因此也是信息时代的安全基石。
   - 没有公钥密码学，HTTPS、数字签名、区块链都不存在。

**这一段写完后读者应该理解**：群论不是学校里学的抽象概念，而是每天在用的技术背后的依赖。

**参考资料**：
- GTIC p2 开头：标准密码学模型（Alice/Bob、对称 vs 非对称）——为"安全基础设施"这个陈述提供权威支撑
- AOGTICA p1 Abstract：对称/非对称的分类框架——说明群论同时用于两种密码

## 第二段：本文路线图（~70 words）

**这一段要传达的核心信息**：本文的完整结构。

**讲解顺序**：

1. 说明本文的目的：面向具备基础数论和集合论知识的高中生读者，介绍常用加密算法的群论原理。
   - 明确读者定位——让读者知道这篇文章是为他们写的。

2. 列出论文结构，直接对应 docx 的三节：
   - Section 1: Cryptographic Algorithms —— 讲群论难题（DHP/DLP/CSP）和各算法的数学原理
   - Section 2: Properties and Applications —— 对比各算法的复杂度、安全性、应用场景
   - Section 3: Testing —— 通过实验验证理论（哈希碰撞、参数不当攻击）
   - Conclusion —— 总结 + 量子威胁 + 展望

3. 说明全文的组织逻辑：
   - 原理 → 算法 → 分析 → 实验 → 总结
   - 让读者预期到每一步做什么

**这一段写完后读者应该理解**：这篇文章要讲什么、怎么讲、最后会得到什么结论。

**参考资料**：
- 无外部 PDF 引用——这一段的路线图是论文自指的

## 写作要点提醒

1. **不要**以"随着互联网的发展……"开头。直接切入"群论与密码学的关系"。
2. **不要**在引言中定义任何群论概念（循环群、生成元等）。这些在 §1 中当场定义。
3. 第一段的"安全基础设施"论点需要与结论最后一段呼应——结论说"理解基础设施才能判断它是否安全"，形成闭环。
4. 引用现在不需要标记——初稿写完后统一添加编号引用。
