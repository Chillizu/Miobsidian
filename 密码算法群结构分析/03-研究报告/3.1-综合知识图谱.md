## MD4 碰撞详细分析

### 算法信息

| 属性 | 值 |
|---|---|
| **算法** | MD4 (Rivest, 1990) |
| **输出长度** | 128-bit |
| **轮数** | 3 轮，每轮 16 步 |
| **构造** | Merkle-Damgård 迭代 |

### 攻击信息

| 属性 | 值 |
|---|---|
| **攻击方法** | 差分攻击 |
| **文献** | Wang et al. (2005) |
| **攻击原理** | 差分路径控制 |
| **攻击过程** | 分析消息差异在压缩函数中的传播 → 引导消息差异相互抵消 → 构造碰撞对 |

### 碰撞结果

- **输入**: 两个视觉上完全不同的输入
- **输出**: MD4 哈希值完全相同
- **复杂度**: 秒级（极低）
- **状态**: 已破

### 安全弱点

1. **轮数少**: 仅 3 轮
2. **非线性函数简单**: 差分路径容易构造
3. **Merkle-Damgård 构造**: 存在长度扩展攻击等弱点

### 群论视角

- MD4 不基于群结构
- 压缩函数使用 $(\mathbb{Z}/2^{32}\mathbb{Z}, +)$ 模加和位运算
- 安全来自混淆与扩散，而非群困难问题

## SHA-1 碰撞详细分析

### 算法信息

| 属性 | 值 |
|---|---|
| **算法** | SHA-1 (NIST, 1995) |
| **输出长度** | 160-bit |
| **轮数** | 80 轮 |
| **构造** | Merkle-Damgård 迭代 |

### 攻击信息

| 属性 | 值 |
|---|---|
| **攻击方法** | SHAttered |
| **文献** | Google + CWI (2017) |
| **攻击复杂度** | ~$2^{63}$ 次 SHA-1 计算 |
| **等效计算** | ~110 GPU-年 |

### 碰撞结果

- **输入**: 两个不同 PDF
- **输出**: SHA-1 值完全相同
- **复杂度**: ~$2^{63}$
- **状态**: 已破

### 对 Git 的影响

- Git 使用 SHA-1 标识提交
- 理论上有风险
- 实践中被利用极难（需要匹配 Git 对象格式）

### 与 MD4 对比

| 属性 | MD4 | SHA-1 |
|---|---|---|
| 输出长度 | 128-bit | 160-bit |
| 轮数 | 3 轮 | 80 轮 |
| 碰撞复杂度 | 秒级 | ~$2^{63}$ |
| 攻击难度 | 简单 | 较复杂但仍可行 |

## SHA-256 安全分析

### 算法信息

| 属性 | 值 |
|---|---|
| **算法** | SHA-256 |
| **输出长度** | 256-bit |
| **轮数** | 64 轮 |
| **构造** | Merkle-Damgård 迭代 |

### 抵抗原因

1. **消息扩展**: 16 blocks → 64 words，放大微小差异
2. **非线性函数**: $\Sigma_0, \Sigma_1, \sigma_0, \sigma_1, \mathrm{Maj}, \mathrm{Ch}$ 混合更充分
3. **轮数**: 更多、设计更复杂

### 碰撞复杂度

- **经典**: ~$2^{128}$（生日攻击）
- **量子**: ~$2^{85}$（Grover）

### 安全状态

- **当前**: 安全下限
- **实际**: 碰撞攻击不可行

| SHA-1 | $O(2^{63})$（~110 GPU-年） | $O(2^{32})$（Grover） |

## 理论背景（§1）

### §1.2.2 哈希函数群论背景

**来源**: GTIC p8

- **Merkle-Damgård 构造**: 输入分块 → 压缩函数迭代
- **非群结构**: 这类哈希函数不直接基于群上的困难问题
- **安全依赖**: 压缩函数的抗碰撞性质

### §1.2.3 哈希函数安全属性

**来源**: AOGTICA p5

- **安全属性**: 原像抵抗、第二原像抵抗、碰撞抵抗
- **安全复杂度**: 原像/第二原像 $O(2^n)$，碰撞 $O(2^{n/2})$（生日攻击）

## 量子威胁（Conclusion）

### 经典攻击 vs 量子攻击

| 算法 | 经典攻击 | 量子攻击 |
|---|---|---|
| MD4/SHA-1 | 经典碰撞攻击 | — |
| SHA-256 | 生日攻击 $O(2^{128})$ | Grover $O(2^{85})$ |
| AES-128 | 穷举 $O(2^{128})$ | Grover $O(2^{64})$ |

### 后量子密码

- **动机**: 经典攻击展示算法设计缺陷
- **方向**: 哈希签名、格密码等非交换/非群结构
- **自然引出**: 从经典攻击到量子威胁

## 写作提示

### 3.1.1 MD4 碰撞

1. **设置**: MD4（128-bit，3 轮），Wang et al. 差分攻击
2. **结果**: 碰撞成立，两个输入 MD4 哈希值相同
3. **分析**: 差分路径控制、轮数少、非线性函数简单
4. **过渡**: "MD4 的脆弱性让密码学家设计了更强的 MD5。"

### 3.1.2 SHA-1 碰撞

1. **设置**: SHA-1（160-bit，80 轮），SHAttered 攻击
2. **结果**: 碰撞存在，两个 PDF SHA-1 值相同
3. **分析**: SHA-1 比 MD4 难但终究被碰、对 Git 的影响
4. **过渡**: "哈希碰撞展示了算法设计缺陷的后果。"

## 注意事项

1. **不展示实际碰撞消息对**: 太长，仅陈述"碰撞已产生"并引用公开结果
2. **强调 MD4 太弱**: 只需要几秒钟就能碰撞
3. **强调 SHA-1 风险**: 虽然碰撞成本高于 MD4，但对于有资源的攻击者是可行的
4. **过渡到 §3.2**: "哈希碰撞展示了算法设计缺陷的后果。类似的原理也适用于非对称密码。"
